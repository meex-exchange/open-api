Open-Api Document

Basic information

API Basic information

This article provides the base URL of the REST interface as https://openapi.meex.com.

All interfaces will return data in JSON format, either as an object or an array.

If the response includes an array, the array elements will be arranged in reverse chronological order, with the earlier data appearing earlier in the array.

All times and timestamps will be provided in UNIX time, measured in milliseconds.

HTTP Response code

HTTP 4XX error codes are used to indicate errors related to the content, behavior, or format of the request.

HTTP 429 is an error code that indicates that the access frequency limit has been reached, and the IP address may be temporarily banned.

HTTP 418 indicates that the request should continue to be made after receiving a 429 error, but doing so may result in being banned.

HTTP 5XX error codes indicate internal server errors, indicating that the problem is on the server side. When dealing with these errors, it is important not to treat them as failed tasks, as the execution status is unknown and could be either a success or a failure.

HTTP 504 indicates that the API server has submitted a request to the business core but has failed to obtain a response. It is important to note that code 504 does not represent a request failure, but rather an unknown status. It is highly likely that the execution has already been completed, or it may have failed and further confirmation is needed.

Any interface may return an error. The incorrect return payload is as follows:

{
    "retCode":1121,
    "retMsg":"Invalid symbol."
    "data":{}
}

Access Restrictions

Each interface will have a description of the frequency limitations for accessing it.

If the frequency restrictions are violated, an HTTP 429 error code will be returned as a warning.

When receiving a 429 warning, the caller should reduce the access frequency or stop accessing the interface altogether.

All interfaces can be accessed up to 300 times per second.

Interface General Information

All requests must use the HTTPS protocol, and the Content-Typerequest header must be set to application/json.

For interfaces using the GET method, parameters must be sent in the query string.

For interfaces using the POST method, parameters must be sent in the request body.

There are no requirements for the order of parameters.

Interface authentication type

  • Each interface has its own authentication type, which determines the required authentication method for access.
  • If an API key is required, it should be passed in the HTTP header as an X-CH-APIKEY field. Both the API key and API secret are case sensitive.
  • API key permissions can be modified in the web user center. For example, permissions may include reading account information, sending transaction instructions, and sending withdrawal instructions.

Interface that requires signature (TRADE and USER_DATA)

  • When calling the TRADE or USER_DATA interface, signature parameters should be passed in the HTTP header as X-CH-SIGN fields.
  • The signature uses the HMAC SHA256 algorithm, with the API secret corresponding to the API key serving as the key for HMAC SHA256
  • The X-CH-SIGN request header takes timestamp+method+requestPath+body string (+ represents string concatenation) as the operation object.
  • The timestamp value is the same as the X-CH-TS request header, with the method being the request method and all uppercase letters: GET/POST.
  • The RequestPath is the path of the requested interface, for example: /sapi/v1/order?orderId=211222334&symbol=btcusdt.
  • The Body is the string of the requested body, which is only applicable for POST requests. For GET requests, the body can be omitted.
  • The signature is case insensitive.

Time synchronization security

  • When signing an interface, a timestamp must be passed in the HTTP header using the X-CH-TS field, which should be the Unix timestamp in milliseconds at the time the request was sent. For example, 1528394129373.
  • When the server receives a request, it will determine the timestamp in the request. If the request was sent more than the specified time window (customizable with the optional parameter recvWindow) before the server receives it, the request will be considered invalid.
  • In addition, if the server calculates that the client timestamp is more than one second in the "future" of the server time, the request will also be rejected.

The following is the logical pseudocode:

if (timestamp < (serverTime + 1000) && (serverTime - timestamp) <= recvWindow) {
  // process request
} else {
  // reject request
}

Regarding transaction timeliness

Internet connectivity is not 100% reliable, and delays from your local connection to the exchange server may fluctuate. To address this issue, the recvWindow parameter can be set.

If you are engaged in high-frequency trading and have strict requirements for transaction timeliness, you can adjust the recvWindow parameter to meet your needs. However, it is not recommended to set recvWindow for more than 5 seconds.

Signature

Sample

/**
 * Generate sign
 **/
private String toSign(String timestamp, String method, String requestPath,
                             String queryString, String body, String secretKey)
                             throws Exception {
    // Signature string
    String preHash = preHash(timestamp, method
    , requestPath, queryString, body);
    byte[] secretKeyBytes = secretKey.getBytes("UTF-8");
    SecretKeySpec secretKeySpec =
    new SecretKeySpec(secretKeyBytes, "HmacSHA256");
    Mac mac = (Mac) MAC.clone();
    mac.init(secretKeySpec);
    return Hex.encodeHexString(mac.doFinal(preHash.getBytes("UTF-8")));
}

/**
 * Signature string
 **/
private String preHash(String timestamp, String method, String requestPath,
String queryString, String body) {

    StringBuilder preHash = new StringBuilder();
    preHash.append(timestamp);
    preHash.append(method.toUpperCase());
    preHash.append(requestPath);
    if (org.apache.commons.lang3.StringUtils.isNotEmpty(queryString)) {
        preHash.append("?").append(queryString);
    }
    if (org.apache.commons.lang3.StringUtils.isNotEmpty(body)) {
        preHash.append(body);
    }
    return preHash.toString();
}

/**
 * queryString
 **/
private String queryString(ServerHttpRequest request) {
    String url = request.getURI().toString();
    String queryString = "";
    if (url.contains("?")) {
        queryString = url.substring(url.lastIndexOf("?") + 1);
    }
    return queryString;
}
