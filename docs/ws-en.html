Open-Api-WebSocket Document

Basic Information

Overview

WebSocket is a new protocol in HTML5. It enables full-duplex communication between the client and the server, allowing data to be quickly transmitted in both directions. A client-server connection can be established through a simple handshake, and the server can proactively push information to the client based on business rules. Its advantages are as follows:

  • When transmitting data between the client and the server, the header information is relatively small, about 2 bytes.
  • Both the client and the server can actively send data to each other.
  • There is no need to create and destroy TCP requests multiple times, saving bandwidth and server resources.

It is highly recommended for developers to use the WebSocket API to access market quotes, trading depth, and other information.

Basic Information

  • Base site for spot market quotes.
  • The returned data is in JSON format.

Subscription Addresses:

Public channels do not require authentication. The following content is only applicable to private channels and subscriptions. When establishing a connection, create an authentication request.

IP Throttling

  • Do not attempt to establish and disconnect connections frequently.
  • Do not establish more than 10 connections within 5 minutes.

Rate Limit

Only for private channels:

  • A maximum of 10 websocket private topic connections per IP simultaneously.

SIGNED Request Example (HMAC)

Here is a step-by-step guide on how to sign a request using the HMAC secret key. Example API key and secret key:

KeyValue
apiKeyvmPUZE6mv9SD5VNHk4HlWFsOr6aKE2zvsw0MuIgwCIPy6utIco14y7Ju91duEh8A
secretKeyNhqPtmdSJYdKjVHjA7PZj4Mge3R5YNiP1e3UZjInClVN65XAbvqqM6A7H5fATj0j

Warning: Do not share your private key with anyone.

Step 1: Create the signature content

Get all request params except for the signature, then sort them in alphabetical order by name:

ParametersExample
apiKeyvmPUZE6mv9SD5VNHk4HlWFsOr6aKE2zvsw0MuIgwCIPy6utIco14y7Ju91duEh8E
timestamp1645423376532

Format the parameters as parameter=value pairs, and separate each parameter with an &.

Step 2: Calculate the signature

  1. Interpret the secretKey as ASCII data and use it as the key for HMAC-SHA-256.
  1. Sign the signature payload as ASCII data.
  1. Encode the HMAC-SHA-256 output as a hex string.

Note that apiKey, secretKey, and payload are case-sensitive. The generated signature value is case-insensitive. You can use OpenSSL to cross-check your algorithm implementation: For example, secretKey=secretKey123455

$ echo -n "1645423376532GET/ws| openssl dgst -sha256 -hmac "secretKey123455"

Step 3: Complete request:

// request
{
    "reqId": "customizationId100001", // optional parameter
    "op": "auth",
    "args": [
        "vmPUZE6mv9SD5VNHk4HlWFsOr6aKE2zvsw0MuIgwCIPy6utIco14y7Ju91duEh8A",
        1662350400000,
        "singature"
    ]
}

// response
{
    "connId":"db94ec37-2b6a-4cb9-97e9-d19cec6e16a6",
    "req":{
        "args":[
            "cdbada06eecbd54fa0127702afb7819e",
            "1684904306427",
            "dd82084ab69d076557d3eb8dd6af31b40f993f77990b9cfb64ad78682219747a"
        ],
        "op":"auth",
        "reqId":"customizationId100001"
    },
    "retCode":"0",
    "retMsg":"Success"
}

Python Example

def auth(self):
    current = str(round(time.time() * 1000))
    auth_request = {
        "reqId": "customizationId100001",
        "op": "auth",
        "args": [
            apikey,
            current,
            self.toSign(current,"GET","/ws","","",secret)
        ]
    }
    auth_request_json = json.dumps(auth_request)
    self.ws.send(auth_request_json)

# generate signature
def toSign(self,timestamp, method, requestPath, queryString, body, secretKey):
    return hmac.new(secretKey.encode('UTF-8'),
                    self.preHash(timestamp, method, requestPath, queryString, body).encode('UTF-8'),
                    "SHA256").hexdigest()

How to subscribe to a topic

The args in the WebSocket

Subscribe to specific topics by passing in args.

// subscribe topic
{
    "reqId": "customizationId100001", // optional parameter
    "op": "subscribe",
    "args": [
        "depth.btcusdt.100"
    ]
}

Multiple topics or symbols can be subscribed to by separating them with commas.

{
    "reqId": "customizationId100001", // optional parameter
    "op": "subscribe",
    "args": [
        "depth.btcusdt.100",
        "ticker.btcusdt"
    ]
}

How to unsubscribe

You can dynamically stop subscribing by sending a request:

// Request
{
    "op": "unsubscribe",
    "args": [
        "depth.btcusdt.100"
    ],
    "reqId": "customizationId100001"
}
// Response
{
    "connId":"db94ec37-2b6a-4cb9-97e9-d19cec6e16a6",
    "req":{
        "op":"unsubscribe",
        "reqId":"reqid0001",
        "topic":"depth.btcusdt.100"
    },
    "retCode":"0",
    "retMsg":"Success"
}x

Response after successful subscription

Example response after a message is pushed
