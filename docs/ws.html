基本資訊

概述

WebSocket是HTML5一种新的協議（Protocol）。它實現了客戶端与服務器全雙工通信， 使得數據可以快速地雙向傳播。通過一次簡單的握手就可以建立客户端和服務器的連接， 服務器根據業務規則可以主動推送信息给客户端。其優點如下：

  • 客戶端和服務器进行數據傳輸時，請求頭信息比較小，大概2個字節。
  • 客户端和服務器皆可以主動地發送數據給對方。
  • 不需要多次創建TCP請求和銷毀，節約寬帶和服務器的資源。

強烈建議開發者WebSocket API獲取市場行情和買賣深度等信息。

SIGNED 请求示例 (HMAC)

這是有關如何用 HMAC secret key 簽署請求的分步指南。

示例API key 和 secret key：

KeyValue
apiKeyvmPUZE6mv9SD5VNHk4HlWFsOr6aKE2zvsw0MuIgwCIPy6utIco14y7Ju91duEh8A
secretKeyNhqPtmdSJYdKjVHjA7PZj4Mge3R5YNiP1e3UZjInClVN65XAbvqqM6A7H5fATj0j

警告：請勿與任何人共享您的私鑰。

第一步：創建簽名內容

除了 signature 之外，獲取所有請求 params，然後按名稱按字母順序進行排序：

参数取值
apiKeyvmPUZE6mv9SD5VNHk4HlWFsOr6aKE2zvsw0MuIgwCIPy6utIco14y7Ju91duEh8A
timestamp1645423376532

將參數格式化為 参数=取值 對，並用 & 分隔每个参数。

第二步：計算簽名

  1. 將 secretKey 解釋為 ASCII 数据，將其用作 HMAC-SHA-256 的 key。
  1. 將 签名有效负载簽名为 ASCII 數據。
  1. 將 HMAC-SHA-256 輸出編碼为 hex string。

請注意，apiKey，secretKey 和有效负载是大小写敏感的。 生成的簽名值是不區分大小寫的。

可以使用 OpenSSL 交叉檢查您的算法實現：

例如secretKey=secretKey123455

$ echo -n "1645423376532GET/ws| openssl dgst -sha256 -hmac "secretKey123455"

第三步：完整請求：

// 请求
{
    "reqId": "customizationId100001", // 可選
    "op": "auth",
    "args": [
        "vmPUZE6mv9SD5VNHk4HlWFsOr6aKE2zvsw0MuIgwCIPy6utIco14y7Ju91duEh8A",
        1662350400000, //過期時間應當大於當前時間戳
        "singature"
    ]
}

// 響應
{
    "connId":"db94ec37-2b6a-4cb9-97e9-d19cec6e16a6",
    "req":{
        "args":[
            "cdbada06eecbd54fa0127702afb7819e",
            "1684904306427",
            "dd82084ab69d076557d3eb8dd6af31b40f993f77990b9cfb64ad78682219747a"

        "op":"auth",

python代碼樣例

def auth(self):
    current = str(round(time.time() * 1000))
    auth_request = {
        "reqId": "customizationId100001",
        "op": "auth",
        "args": [
            apikey,
            current,
            self.toSign(current,"GET","/ws","","",secret)
        ]
    }
    auth_request_json = json.dumps(auth_request)
    self.ws.send(auth_request_json)

# generate signature
def toSign(self,timestamp, method, requestPath, queryString, body, secretKey):
    return hmac.new(secretKey.encode('UTF-8'),
                    self.preHash(timestamp, method, requestPath, queryString, body).encode('UTF-8'),
                    "SHA256").hexdigest()

如何訂閱topic

Websocket裡的args

通過傳入args來訂閱指定topic

// 訂閱topic
{
    "reqId": "customizationId100001", // 可選
    "op": "subscribe",
    "args": [
        "depth.btcusdt.100"
    ]
}

通過逗號隔開，可以同時訂閱多個topic或者多個symbol

{
    "reqId": "customizationId100001", // 可選
    "op": "subscribe",
    "args": [
        "depth.btcusdt.100",
        "ticker.btcusdt"
    ]
}

如何取消訂閱

您可以通過發送請求來動態地停止訂閱:

// 请求
{
    "op": "unsubscribe",
    "args": [
        "depth.btcusdt.100"
    ],
    "reqId": "customizationId100001"
}
// 響應
{
    "connId":"db94ec37-2b6a-4cb9-97e9-d19cec6e16a6",
    "req":{
        "op":"unsubscribe",
        "reqId":"reqid0001",
        "topic":"depth.btcusdt.100"
    },
    "retCode":"0",
    "retMsg":"Success"
}

訂閱的響應

訂閱成功後的響應示例
{
    "connId":"db94ec37-2b6a-4cb9-97e9-d19cec6e16a6",
    "req":{
        "op":"subscribe",
        "reqId":"customizationId100001",
        "topic":"depth.btcusdt.100"

推送頻率：2s

档位：1、5、100

    "op": "subscribe",
    "args": [
        "depth.btcusdt.1"
    ]
}

tsstring消息被推送出來的時間戳 (毫秒)
dataObjectObject
> sstring交易對

>barrayBid, 買方. 按照價格從大到小
>> b[0]string買方報價
>> b[1]string買方數量
> aarrayAsk, 賣方. 按照價格從小到大
>> a[0]string賣方報價
>> a[1]string賣方數量
