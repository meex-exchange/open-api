Meex Open Api

基本資訊

API基本資訊

所有接口都會返回一個JSON object或者array

響應中如有數組，數組元素以時間倒序排列，越早的數據越提前

所有時間、時間戳記均為UNIX時間，組織為毫秒

HTTP返回程式碼

HTTP 4XX 錯誤碼用於訓示錯誤的請求內容、行為、格式

HTTP 429 錯誤碼表示警告訪問頻次超限，即將被封IP

HTTP 418 表示收到429後繼續訪問，於是被封了

HTTP 5XX 返回錯誤碼是內部系統錯誤； 這說明這個問題是在服務器這邊

在對待這個錯誤時，千萬不要把它當成一個失敗的任務，因為執行狀態未知，有可能是成功也有可能是失敗

HTTP 504 表示API服務端已經向業務覈心提交了請求但未能獲取響應，

特別需要注意的是504程式碼不代表請求失敗，而是未知

很可能已經得到了執行，也有可能執行失敗，需要做進一步確認.

任何接口都可能返回ERROR（錯誤）； 錯誤的返回payload如下：

{
    "retCode":1121,
    "retMsg":"Invalid symbol."
    "data":{}
}

訪問限制

在每個接口下麵會有限頻的說明.

違反頻率限制都會收到HTTP 429，這是一個警告.

當收到429告警時，調用者應當降低訪問頻率或者停止訪問.

所有接口訪問300次/秒

接口通用資訊

所有請求基於Https協定，請求頭資訊中Content-Type

需要統一設定為: 'application/json'

GET方法的接口, 參數必須在query string中發送

POST方法的接口, 參數必須在request body中發送

對參數的順序不做要求。

接口鑒權類型

  • 每個接口都有自己的鑒權類型，鑒權類型决定了訪問時應當進行何種鑒權
  • 如果需要API-key，應當在HTTP頭中以X-CH-APIKEY欄位傳遞
  • API-key與API-secret是大小寫敏感的
  • 可以在網頁用戶中心修改API-key所具有的許可權
  • 例如讀取帳戶資訊、發送交易指令、發送提現指令
鑒權類型描述
NONE不需要鑒權的接口
TRADE需要有效的API-KEY和簽名
USER_DATA需要有效的API-KEY和簽名
USER_STREAM需要有效的API-KEY
MARKET_DATA需要有效的API-KEY

需要簽名的接口 (TRADE 與 USER_DATA)

  • 調用TRADE或者USER_DATA接口時，應當在HTTP頭中以X-CH-SIGN欄位傳遞簽名參數.
  • 簽名使用HMAC SHA256算灋. API-KEY所對應的API-Secret作為HMAC SHA256 的金鑰..
  • X-CH-SIGN的請求頭是以timestamp + method + requestPath + body字串（+表示字串連接）作為操作對象
  • 其中timestamp的值與X-CH-TS請求頭相同, method是請求方法，字母全部大寫：GET/POST.
  • requestPath是請求接口路徑例如：/sapi/v1/order?orderId=211222334&symbol=btcusdt
  • body是請求主體的字串（post only）如果是GET請求則body可省略
  • 簽名大小寫不敏感。

時間同步安全

  • 簽名接口均需要在HTTP頭中以X-CH-TS欄位傳遞時間戳記, 其值應當是請求發送時刻的unix時間戳記（毫秒）e.g. 1528394129373
  • 服務器收到請求時會判斷請求中的時間戳記，如果是5000毫秒之前發出的，則請求會被認為無效。 這個時間視窗值可以通過發送可選參數recvWindow來自定義。
  • 另外，如果服務器計算得出用戶端時間戳記在服務器時間的‘未來’一秒以上，也會拒絕請求。
  • 邏輯偽代碼：
if (timestamp < (serverTime + 1000) && (serverTime - timestamp) <= recvWindow) {
  // process request
} else {
  // reject request
}

關於交易時效性 關於交易時效性互聯網狀況並不100%可靠，不可完全依賴，囙此你的程式本地到交易所服務器的時延會有抖動. 這是我們設定recvWindow的目的所在，如果你從事高頻交易，對交易時效性有較高的要求，可以靈活設定recvWindow以達到你的要求。 不推薦使用5秒以上的recvWindow

簽名

樣例

/**
 * 生成sign
 **/
private String toSign(String timestamp, String method, String requestPath,
                             String queryString, String body, String secretKey)
                             throws Exception {
    // 簽名串
    String preHash = preHash(timestamp, method
    , requestPath, queryString, body);
    byte[] secretKeyBytes = secretKey.getBytes("UTF-8");
    SecretKeySpec secretKeySpec =
    new SecretKeySpec(secretKeyBytes, "HmacSHA256");
    Mac mac = (Mac) MAC.clone();
    mac.init(secretKeySpec);
    return Hex.encodeHexString(mac.doFinal(preHash.getBytes("UTF-8")));
}

/**
 * 簽名串
 **/
private String preHash(String timestamp, String method, String requestPath,
String queryString, String body) {

    StringBuilder preHash = new StringBuilder();
    preHash.append(timestamp);
    preHash.append(method.toUpperCase());
    preHash.append(requestPath);
    if (org.apache.commons.lang3.StringUtils.isNotEmpty(queryString)) {
        preHash.append("?").append(queryString);
    }
    if (org.apache.commons.lang3.StringUtils.isNotEmpty(body)) {
        preHash.append(body);
    }
    return preHash.toString();
}

/**
 * queryString
 **/
private String queryString(ServerHttpRequest request) {
    String url = request.getURI().toString();
    String queryString = "";
    if (url.contains("?")) {
        queryString = url.substring(url.lastIndexOf("?") + 1);
    }
    return queryString;
}

ENUM

術語解釋:

  • base 指一個交易對的交易對象，即寫在靠前部分的資產名
  • quote 指一個交易對的定價資產，即寫在靠後部分資產名

訂單狀態：

  • NEW 新訂單，無成交
  • PART_FILLED 部分成交
  • FILLED 全部成交
  • CANCELED 已取消
  • PENDING_CANCEL 正在撤销中
  • REJECTED 訂單被拒絕

訂單種類：

  • LIMIT 限價單
  • MARKET 市價單

訂單方向：

  • BUY 買單
  • SELL 賣單

K線間隔:

min -> 分鐘; h -> 小時; day -> 天; week -> 周; month -> 月
